# Development Workflow

## Methodology: Controlled Vibe Coding

Code is generated by AI (Claude, GPT-4, etc.), reviewed by the developer, and versioned with git.

**Golden rule:** You don't need to write every line, but you need to understand every line that goes into production.

## Development Cycle

```
1. Pick a task from /docs/CONTEXT.md → checklist
2. Generate code via AI (clear prompt, specifying file and feature)
3. Integrate into the project
4. Run npm run dev → check for compilation errors
5. Test in desktop browser
6. If mobile: npm run dev -- --host, test on phone on the same network
7. If working: git add -A && git commit -m "type: description"
8. If broken: git checkout -- . && go back to step 2 with correction
```

## Commit Convention

Use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` — new feature
- `fix:` — bug fix
- `refactor:` — refactoring without behavior change
- `docs:` — documentation
- `style:` — formatting, CSS
- `perf:` — performance
- `test:` — tests
- `chore:` — build, dependencies, config

## CI/CD

Pushes to `main` trigger GitHub Actions:
1. Builds Docker image (`node:20-alpine` → `nginx:stable-alpine`)
2. Pushes to `ghcr.io/linkzy/ComuNick`
3. Deploys to `https://comunick.linkzy.dev`

## Testing

**Philosophy:** Manual testing on the target device is more important than automated tests at this stage. Priority is validation with the child and speech therapist.

**When to automate:**
- After v1.0.0, if the app gains traction
- Critical tests: IndexedDB CRUD operations, TTS fallback
- Tool: Vitest (already included in Vite ecosystem)

## Common Troubleshooting

| Symptom | Probable Cause | Solution |
|---------|---------------|----------|
| TTS doesn't play on first tap | Warmup wasn't called | Check that TTSEngine.warmup() is called in SplashScreen |
| Grid doesn't update after editing | BoardContext not propagating | Check dispatch UPDATE_BOARD |
| PWA won't install | Missing HTTPS or invalid manifest | Serve via HTTPS, validate manifest.json |
| ARASAAC returns no results | Rate limit or network offline | Check cache, implement retry |
| App breaks on phone but works on desktop | Missing touch event support | Use onClick (works for mouse and touch) instead of onTouchStart |
