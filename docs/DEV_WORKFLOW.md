# Development Workflow

## Methodology: Controlled Vibe Coding

Code is generated by AI (Claude, GPT-4, etc.), reviewed by the developer, and versioned with git.

**Golden rule:** You don't need to write every line, but you need to understand every line that goes into production.

## Initial Setup

```bash
# 1. Initialize project
npm create vite@latest comunick -- --template react
cd comunick
npm install
npm install vite-plugin-pwa idb i18next react-i18next

# 2. Initialize git
git init
git add -A
git commit -m "feat: initial scaffold from Vite template"

# 3. Verify everything runs
npm run dev

# 4. Test on phone (same network)
# npm run dev -- --host
# Access http://{IP}:5173 on phone
```

## Development Cycle

```
1. Pick a task from /docs/CONTEXT.md → checklist
2. Generate code via AI (clear prompt, specifying file and feature)
3. Integrate into the project
4. Run npm run dev → check for compilation errors
5. Test in desktop browser
6. If mobile: npm run dev -- --host, test on phone on the same network
7. If working: git add -A && git commit -m "type: description"
8. If broken: git checkout -- . && go back to step 2 with correction
```

## Commit Convention

Use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` — new feature
- `fix:` — bug fix
- `refactor:` — refactoring without behavior change
- `docs:` — documentation
- `style:` — formatting, CSS
- `perf:` — performance
- `test:` — tests
- `chore:` — build, dependencies, config

## Version Tags

```bash
# Stable versions (tag BEFORE testing with the child)
git tag v0.1.0 -m "Static 2x1 grid working with TTS"
git tag v0.2.0 -m "Admin mode with cell editing"
git tag v1.0.0 -m "First usable version for therapist and child"

# If something breaks, revert:
git checkout v0.1.0
```

## AI Prompts

When generating code with AI, use the following prompt format:

```
File: src/components/Grid.jsx
Feature: Responsive grid that renders cells based on BoardContext
Requirements:
- Uses CSS Grid with grid-template-columns: repeat(cols, 1fr)
- Calculates cell size with min(height/rows, width/cols)
- Loading state: shows SplashScreen
- Empty state (rows=0 or cols=0): shows "Configure the board in admin mode"
- Edge case: rows/cols change should not lose existing cells
- Uses BoardContext, does not receive board props
Style:
- React functional component with hooks
- CSS modules or separate Grid.css file
- No external dependencies
```

## Testing

**Philosophy:** Manual testing on the target device is more important than automated tests at this stage. Priority is validation with the child and speech therapist.

**When to automate:**
- After v1.0.0, if the app gains traction
- Critical tests: IndexedDB CRUD operations, TTS fallback
- Tool: Vitest (already included in Vite ecosystem)

**Pre-publication manual checklist:**
- [ ] Install app via PWA
- [ ] Open offline (airplane mode)
- [ ] Tap cells → hear sound
- [ ] Change language → UI updates + TTS in new language
- [ ] Admin mode: change rows/cols → grid resizes
- [ ] Admin mode: add cell with ARASAAC image → image appears
- [ ] Admin mode: delete board → returns to previous board
- [ ] Rotate screen → grid adjusts
- [ ] 50 cells in grid → no lag

## Common Troubleshooting

| Symptom | Probable Cause | Solution |
|---------|---------------|----------|
| TTS doesn't play on first tap | Warmup wasn't called | Check that TTSEngine.warmup() is called in SplashScreen |
| Grid doesn't update after editing | BoardContext not propagating | Check dispatch UPDATE_BOARD |
| PWA won't install | Missing HTTPS or invalid manifest | Serve via HTTPS, validate manifest.json |
| ARASAAC returns no results | Rate limit or network offline | Check cache, implement retry |
| App breaks on phone but works on desktop | Missing touch event support | Use onClick (works for mouse and touch) instead of onTouchStart |
