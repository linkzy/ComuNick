# Internationalization (i18n)

## i18next Setup

Use i18next with automatic language detection.

```js
// i18n/index.js
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import ptBR from "./pt-BR.json";
import en from "./en.json";
import es from "./es.json";

i18n.use(initReactI18next).init({
  resources: {
    "pt-BR": { translation: ptBR },
    "en": { translation: en },
    "es": { translation: es },
  },
  fallbackLng: "pt-BR",
  detection: {
    // Detect from navigator.language
    order: ["navigator"],
    caches: [], // don't cache for automatic detection
  },
  interpolation: {
    escapeValue: false,
  },
});

export default i18n;
```

## Language Flow

```
1. App opens
2. i18next init with navigator.language (e.g.: "pt-BR", "en", "es")
3. Settings loaded from IndexedDB
4. If settings.lang exists and differs from detected → use settings.lang
5. If user changes manually → save to settings.lang
6. TTS uses i18next language for utterance.lang
```

## Adding a New Language

1. Create `i18n/{code}.json` file
2. Import and add to `i18n/index.js` resources
3. Add entry in the `language` key of each existing language JSON
4. Add entry in the LanguageSelector component

## TTS and i18n

The TTS language must match the i18n language:

```js
TTSEngine.speak(text, i18next.language);
```

Important: `navigator.language` returns "pt-BR", but the BCP 47 tag for TTS may be "pt-BR" (Chrome) or "pt_BR" (some devices). The Web Speech API accepts both, but test on the target device.

## Best Practices

1. **Never hardcode strings** — every user-visible string must use `t()` from i18next
2. **Proper names ("ComuNick", "ARASAAC")** should not be translated
3. **Numbers and formatting** may vary by locale — use the `Intl` API when needed
4. **Automatic detection** does not replace manual choice — always allow switching
5. **Fallback always pt-BR** — if a translation key is missing in a language, show pt-BR
